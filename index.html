Core Principles 01

언어 원칙

모든 UI 문구 작성 시 따라야 하는 기본 원칙입니다.

6가지 핵심 원칙
  • 1사용자 중심 — 시스템이 아닌 사용자 관점에서 작성. "오류 코드 E-404" → "페이지를 찾을 수 없습니다"
  • 2간결하고 명확하게 — 불필요한 단어 제거. 한 문장에 하나의 개념만
  • 3능동태 사용 — "파일이 삭제되었습니다" → "파일을 삭제했습니다"
  • 4구체적 행동 유도 — 다음에 무엇을 해야 하는지 명확하게 안내
  • 5일관된 용어 — 같은 기능·개념은 항상 같은 단어 사용
  • 6긍정적 표현 — "입력하지 않으면 안 됩니다" → "필수로 입력해 주세요"
✓ Do
"인터넷 연결을 확인하고 다시 시도해 주세요."

"이름을 입력해 주세요."

"파일을 저장했습니다."
✗ Don't
"네트워크 에러가 발생했습니다. (err: ETIMEDOUT)"

"이름 필드가 비어 있습니다."

"파일이 저장 완료되었습니다."
Brand Voice 02

브랜드 보이스

KRDS 기반 서비스의 언어적 성격과 일관된 표현 방식입니다.

신뢰감 있고 전문적

정확한 정보와 명확한 안내로 신뢰를 구축합니다.

"본인 확인이 완료되었습니다."
"서류를 검토 중입니다. 영업일 기준 3일 내 안내드립니다."
친근하고 배려 있게

딱딱하지 않고, 사용자를 존중하는 말투를 사용합니다.

"잠깐이면 됩니다. 정보를 확인해 주세요."
"처음 사용하시나요? 단계별로 안내해 드릴게요."
명확하고 직접적

돌려 말하지 않고 핵심을 바로 전달합니다.

"신청 기간이 종료되었습니다. (마감일: 2025. 12. 31)"
"이미 신청한 내역이 있습니다."
해결 중심적

문제를 알리는 것에 그치지 않고 해결책을 함께 안내합니다.

"파일 크기가 초과되었습니다. 10MB 이하로 다시 업로드해 주세요."
Tone & Manner 03

톤앤매너

상황과 맥락에 따라 적절한 톤을 선택해 일관된 사용자 경험을 만듭니다.

상황별 톤 가이드
  • 1일반 안내 — 중립적이고 명확하게. "신청서를 작성해 주세요."
  • 2오류 상황 — 차분하고 해결 중심으로. 사용자 불안을 최소화. "다시 시도해 주세요."
  • 3경고·주의 — 명확하되 위협적이지 않게. "저장하지 않은 내용이 사라집니다."
  • 4성공·완료 — 따뜻하고 긍정적으로. "신청이 완료되었습니다!"
  • 5온보딩 — 친근하고 격려하는 톤. "처음 방문이시군요! 함께 시작해 볼까요?"
  • 6빈 상태 — 행동을 유도하는 긍정적 톤. "아직 내역이 없습니다. 지금 시작해 보세요."
✓ Do
"입력하신 정보를 확인 중입니다."

"신청이 성공적으로 완료되었습니다."

"잠시 후 다시 시도해 주세요."
✗ Don't
"처리 중. 기다리시오."

"신청 완료!!!"

"오류가 발생했습니다. 관리자에게 문의하세요."

Form & Input 05

폼 & 입력

폼 요소의 레이블, 플레이스홀더, 도움말 텍스트를 올바르게 작성해 입력 실수를 줄입니다.

플레이스홀더 원칙
  • 1레이블 대체 금지 — 플레이스홀더는 레이블을 대신할 수 없습니다. 항상 레이블과 함께 사용.
  • 2형식 힌트 전용 — 예시 값이나 형식 안내에만 사용. 예) "예) hong@email.com"
  • 3필수·선택 구분 — 필수 항목은 레이블 옆에 "(필수)" 또는 * 표시. 선택은 "(선택)"
레이블 작성 예시
이름 / 연락처
이름 (필수)
휴대폰 번호 — 예) 010-1234-5678
이메일 / 비밀번호
이메일 주소 — 예) hong@email.com
비밀번호 — 영문·숫자 조합 8자 이상
개인 식별 정보
주민등록번호 앞 6자리 — 이 정보는 본인 확인 목적으로만 사용됩니다.

Accessibility & i18n 09

접근성 & 국제화

KWCAG 2.2 및 WCAG 2.1 AA를 준수해 모든 사용자가 서비스를 이용할 수 있도록 합니다.

aria-label 작성 가이드
  • 1아이콘 버튼 — 시각적 아이콘만 있는 버튼은 aria-label 필수. aria-label="메뉴 열기"
  • 2이미지 — 정보를 전달하는 이미지는 alt 속성 필수. 장식용 이미지는 alt=""
  • 3링크 — "여기를 클릭" 대신 목적지를 명확히. "사용 설명서 PDF 다운로드"
  • 4폼 연결 — label의 for 속성과 input의 id 속성을 반드시 연결
국제화 고려사항
  • 1문자열 외부화 — 하드코딩 대신 i18n 키로 관리. 텍스트 길이 여유 공간 확보 (+30~40%)
  • 2날짜·숫자 형식 — 로케일에 맞는 형식 사용. 한국: 2025년 12월 31일 / 영어: Dec 31, 2025
  • 3문화적 맥락 — 관용어, 속담, 문화 특정 표현 지양. 번역 시 직역보다 의미 전달 중심
