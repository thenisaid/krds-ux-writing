# 대한민국법원 전자가족관계등록시스템 — UX Writing 분석 및 파생 가이드

> **기관**: 대한민국법원 전자가족관계등록시스템 (https://efamily.scourt.go.kr/)
> **분석 기준**: KRDS UX Writing 3대 원칙 (무번역·정보핵심화·심리적안전망)
> **Layer 2 트랙**: 표준형 (사법부 소속 공식 시스템)
> **주요 사용자**: 출생·혼인·이혼·사망 신고자, 가족관계증명서 발급 필요자, 성년후견 신청자
> **작성 기준**: IND-136 | 2026-04-23

---

## 1. 기관 개요 및 서비스 맥락

전자가족관계등록시스템은 출생·혼인·이혼·사망 등 **인생의 주요 사건을 법적으로 등록**하고  
가족관계증명서·기본증명서 등을 발급하는 법원 운영 시스템입니다.

사용자는 기쁘거나(출생·혼인) 슬픈(사망·이혼) 감정적으로 취약한 상태에서 이 시스템을 이용하는 경우가 많습니다. UX Writing에서 **심리적안전망 원칙**이 특히 중요한 기관입니다.

| 서비스 유형 | 대표 과업 |
|------------|---------|
| 등록 신고 | 출생신고, 혼인신고, 이혼신고, 사망신고 |
| 증명서 발급 | 가족관계증명서, 기본증명서, 혼인관계증명서, 입양관계증명서 |
| 입양 관련 | 입양신고, 파양신고 |
| 성년후견 | 후견등기사항증명서 발급 |
| 개명·국적 | 개명신청 안내, 국적 취득·상실 신고 |

## 2. UX Writing 진단 — 3대 원칙 기준

### 2.1 무번역의 원칙 위반 사례

법원 시스템 특성상 법령 용어가 UI에 그대로 노출되는 경우가 많습니다.  
특히 사용자 상태(출생 직후 신고, 사별 직후 사망신고)를 고려한 배려 언어가 필요합니다.

| 현재 표현 | 문제 | 권고 표현 |
|---------|------|---------|
| 신고의무자 | 법령 용어 | 신고해야 하는 분 |
| 등록기준지 | 행정 전문 용어 (구 호적 본적지 개념) | 가족관계 기준 주소 (예: 부모의 등록기준지) |
| 가족관계등록부 | 법령 용어 | 가족관계 기록 |
| 기재사항 | 행정어 | 기록된 내용 |
| 친생자 | 법률 용어 | 직접 낳은 자녀 |
| 인지 | 법률 용어 (혼외 자녀 법적 인정) | 법적 부모 확인 신청 |
| 파양 | 법률 용어 | 입양 취소 |
| 후견인 | 법률 용어 | 법정 대리인 |
| 성년후견 | 법률 용어 | 판단 능력이 없는 성인을 위한 법적 보호 |
| 피후견인 | 법률 용어 | 후견을 받는 분 |
| 특정후견 | 법률 용어 | 특정 사무를 위한 후견 |
| 협의이혼 | 법률 용어 (일상어 수준이나 설명 필요) | 합의 이혼 |
| 재판이혼 | 법률 용어 | 법원에서 결정하는 이혼 |
| 출생신고서 접수 | 행정어 | 출생신고 제출 |
| 첨부서류 | 행정어 | 필요한 서류 |

**법원 시스템 특이 사항**:  
법령에 명시된 신고 유형명(출생신고, 혼인신고, 사망신고 등)은 원어를 유지합니다.  
단, 첫 등장 시 괄호 병기 또는 도움말(ⓘ)로 일상어 설명을 추가합니다.

---

### 2.2 정보핵심화의 원칙 위반 사례

**발견 위치**: 신고 안내, 필요 서류 목록, 처리 결과 화면

| 현재 표현 | 문제 | 권고 방향 |
|---------|------|---------|
| 출생신고 안내: "출생신고는 출생일로부터 1개월 이내에 신고의무자가 신고하여야 하며, 기간 내 신고하지 않을 경우 과태료가 부과될 수 있습니다." | 핵심(신고 기한)이 설명 중간에 묻힘 | "출생일로부터 1개월 안에 신고해야 합니다. [신고하기]" |
| 필요 서류 목록: 8~10개 항목을 번호 없이 나열 | 항목이 많아 무엇을 먼저 준비해야 하는지 불명확 | 필수 서류 / 선택 서류로 분리 |
| 가족관계증명서 발급 완료: "증명서 발급이 완료되었습니다. 보안상의 이유로 10분 내에 출력하여 주시기 바랍니다." | 핵심(10분 제한)이 부드러운 표현 뒤에 있어 놓치기 쉬움 | "10분 안에 출력해 주세요. (보안 정책으로 시간이 지나면 다시 발급해야 합니다)" |

---

### 2.3 심리적안전망의 원칙 위반 사례

**발견 위치**: 신고 오류, 서류 미비 안내, 사망신고 관련 화면

| 현재 표현 | 문제 | 권고 방향 |
|---------|------|---------|
| "입력하신 내용에 오류가 있습니다. 확인 후 다시 입력해 주세요." | 어느 항목이 오류인지 불명확 | 오류 항목 강조 + 올바른 입력 형식 예시 제공 |
| "서류가 불충분합니다." | 어떤 서류가 왜 부족한지 불명확 | "○○ 서류가 누락됐습니다. 발급 방법: [○○○○ 발급처 안내]" |
| 사망신고 오류 화면: "사망진단서 첨부가 필요합니다." | 슬픔 상태의 사용자에게 무미건조한 에러 | "사망진단서가 필요합니다. 병원에서 발급받을 수 있습니다." — 동일 내용이나 발급처 안내 추가 |

**감정 배려 원칙 (이 기관 전용)**:  
사망신고·이혼신고 화면에서 `~하십시오`식 명령형 대신 `~해 주세요`식 안내형을 사용합니다.  
오류 메시지에서 "잘못", "틀린", "오류" 표현은 사용자 탓처럼 느껴질 수 있으므로  
"맞지 않습니다", "확인이 필요합니다"로 대체합니다.

## 4. Before / After 개선 예시

### E1 — 출생신고 기한 안내

```
🚫 현재:
"출생신고는 출생일로부터 1개월 이내에 신고의무자가 신고하여야 하며,
기간 내 신고하지 않을 경우 과태료가 부과될 수 있습니다."

✅ 개선:
출생일로부터 1개월 안에 신고해야 합니다.
기한을 넘기면 과태료가 부과될 수 있습니다.
[출생신고 바로 하기]

신고해야 하는 분: 부모 중 한 명 (부모가 없는 경우 동거인 또는 분만을 도운 의사·조산사)
```

> 적용 원칙: 정보핵심화 (기한 첫 줄), 무번역 (신고의무자 → 신고해야 하는 분)

---

### E2 — 가족관계증명서 발급 완료

```
🚫 현재:
"증명서 발급이 완료되었습니다. 보안상의 이유로 10분 내에 출력하여 주시기 바랍니다."

✅ 개선:
가족관계증명서 발급이 완료됐습니다.
10분 안에 출력해 주세요.
시간이 지나면 보안을 위해 자동으로 닫힙니다. 다시 발급하면 됩니다.
[지금 출력하기]  [PDF로 내려받기]
```

> 적용 원칙: 정보핵심화 (10분 기한 강조), 무번역 (명령형 → 안내형), 심리적안전망 (기한 초과 시 재발급 가능 안내)

---

### E3 — 사망신고 서류 미비

```
🚫 현재:
"서류가 불충분합니다. 다시 확인하여 주십시오."

✅ 개선:
사망진단서가 필요합니다.
사망이 확인된 병원에서 발급받을 수 있습니다.
서류를 준비한 뒤 다시 신고해 주세요.
도움이 필요하면 관할 가정법원(또는 시·군·구청 담당자)에 문의해 주세요.
```

> 적용 원칙: 심리적안전망 (서류 발급처 안내, 사용자 탓 없음, 감정 배려), 무번역 (불충분 → 누락 서류 특정)

---

### E4 — 이름(개명) 신청 입력 오류

```
🚫 현재:
"입력 오류입니다. 올바른 이름을 입력해 주세요."

✅ 개선:
이름 입력 내용을 확인해 주세요.
한글 또는 한자로만 입력할 수 있으며, 특수문자는 사용할 수 없습니다.
예: 홍길동 / 洪吉童
```

> 적용 원칙: 심리적안전망 (원인 + 예시 제시, 사용자 탓 없음), 정보핵심화 (입력 규칙 구체화)

---

### E5 — 등록기준지 미입력 (폼 레이블 개선)

```
🚫 현재:
레이블: "등록기준지"  플레이스홀더: ""

✅ 개선:
레이블: "가족관계 기준 주소 (등록기준지)"
도움말 ⓘ: 혼인신고 당시 설정한 부모의 주민등록상 주소입니다. 모를 경우 가족관계증명서에서 확인하세요.
플레이스홀더: 예: 서울특별시 종로구 (빈칸으로 두면 부/모의 등록기준지를 따릅니다)
```

> 적용 원칙: 무번역 (등록기준지 일상어 병기), 정보핵심화 (모를 경우 확인 방법 안내)

## 5. 기관 맞춤 톤·매너 권고

### 톤 기준

| 상황 | 권고 톤 | 샘플 |
|------|---------|------|
| 신고 안내 | T2 길잡이 + 안심 | "다음 항목을 차례대로 작성해 주세요." |
| 발급 완료 | T4 마침표 (절제) | "가족관계증명서 발급이 완료됐습니다." |
| 오류·서류 미비 | T1 해결사 + 감정 배려 | "○○ 서류가 필요합니다. ○○에서 발급받을 수 있습니다." |
| 처리 대기 | T3 중계자 | "신고 내용을 검토하고 있습니다. 처리까지 3~5 영업일이 걸립니다." |

### 전자가족관계등록시스템 전용 규칙

1. **법적 사건 존중**: 출생·혼인·사망·이혼 화면에서 감정적 배려 표현 추가 (예: 이혼신고 → "절차를 진행하시는 데 필요한 서류를 안내드립니다")
2. **명령형 금지**: "~하십시오" → "~해 주세요" (안내형 통일)
3. **오류 표현 순화**: "잘못됐습니다" → "맞지 않습니다", "틀렸습니다" → "확인이 필요합니다"
4. **서류 발급처 링크**: 미비 서류 안내 시 발급처·방법 반드시 포함
5. **시간 제한 강조**: 10분 출력 제한 등 시간 제약은 첫 문장에
6. **관할법원 연락처**: 복잡한 사안(성년후견·입양취소 등) 에러 화면에 관할 가정법원 연락처 포함

## 6. 자가진단 체크리스트 — 전자가족관계등록시스템 맞춤

**공통 항목 (Layer 2 상속)**
```
□ 법령 용어 첫 등장 시 괄호 또는 도움말로 일상어 설명이 있는가?
□ 에러 메시지에 사용자를 탓하는 표현이 없는가?
□ 에러 메시지에 해결 방법 또는 문의 연락처가 포함됐는가?
□ 버튼 레이블이 동사형 또는 관용적 허용어인가?
□ 결론이 첫 문장에 있는가?
□ 명령형 어조("~하십시오")가 없는가?
```

**전자가족관계등록시스템 전용 항목**
```
□ 신고 기한(출생 1개월 등)이 안내 첫 문장에 명시됐는가?
□ 서류 미비 시 해당 서류의 발급처·방법이 안내됐는가?
□ 발급 완료 화면에 출력 제한 시간이 명확히 표시됐는가?
□ 사망·이혼 신고 화면에서 감정 배려 표현을 사용했는가?
□ 오류 표현이 "잘못됐습니다" 대신 "맞지 않습니다" 형식인가?
□ 복잡한 사안(후견·입양취소)에 관할 가정법원 연락처가 있는가?
□ 등록기준지 등 이해하기 어려운 법적 개념에 도움말(ⓘ)이 있는가?
```

### 7.2 실사 기반 Before / After — 추가

#### B1 — 영문 단독 버튼 (무번역 원칙 최우선 적용)

```
🚫 현재:
버튼: "Verification of Certificate Authenticity"
(한국어 UI 내 영문만 표기)

✅ 개선:
버튼: "증명서 진위확인"
필요 시 병기: "증명서 진위확인 (Verify Certificate)"
```

> 적용 원칙: 무번역 (한국어 서비스에 영문 단독 UI 금지), 접근성 (스크린리더 한국어 TTS 처리 불가 문제 해소)

---

#### B2 — 섹션명 자연어 교정

```
🚫 현재:
"자주하는질문"

✅ 개선:
"자주 묻는 질문"
```

> 적용 원칙: 무번역 (자연어 미사용 교정) — 단순 오탈자 수준이나 공신력 있는 기관에서 방치되어 있는 사례

---

#### B3 — 법률 서비스명 → 사용자 과업 중심 명칭

```
🚫 현재:
메뉴: "가족관계등록부정정"

✅ 개선:
메뉴: "가족관계 오류 수정 신청"
도움말 ⓘ: 잘못 기재된 가족관계 정보(이름, 생년월일 등)를 수정 신청합니다.
```

> 적용 원칙: 무번역 (법률 용어 → 사용자 과업 언어), 심리적안전망 (도움말로 진입 장벽 낮춤)

---

#### B4 — 귀화자 대상 법률 서비스명

```
🚫 현재:
"국적취득자의 성본창설"

✅ 개선:
"귀화 후 성씨·본관 등록"
도움말 ⓘ: 귀화로 한국 국적을 취득한 분이 성씨와 본관을 새로 등록하는 절차입니다.
```

> 적용 원칙: 무번역 (법률 용어 → 사용자 상황 언어), 심리적안전망 (대상자 명확화)

---

```
✅ 개선:
"구 서류 스캔 등본 (이미지제적)"
도움말 ⓘ: 전산화 이전(2008년 이전) 종이 제적부를 스캔한 파일로 발급하는 등본입니다.
```

> 적용 원칙: 무번역 (전문 용어에 일상어 병기), 심리적안전망 (언제 필요한지 맥락 설명)
