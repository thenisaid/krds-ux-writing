## 1. 기관 개요 및 서비스 맥락

정부24는 3,000여 개 민원·행정 서비스의 단일 진입점입니다.  
주요 서비스:

| 서비스 유형 | 대표 과업 |
|------------|---------|
| 증명서 발급 | 주민등록등·초본, 가족관계증명서, 건강보험료 납부확인서 |

| 본인인증 | 공동인증서, 간편인증, 문자인증 |

## 2. UX Writing 진단 — 3대 원칙 기준

### 2.1 무번역의 원칙 위반 사례

**발견 위치**: 서비스 분류 메뉴, 안내 문구, 법정 서식 레이블

| 현재 표현 | 문제 | 권고 표현 |
|---------|------|---------|
| 원스톱 서비스 | 외래어, 14세 이하 이해 불가 | 한 번에 처리하는 서비스 |
| 공통기반서비스 | IT 내부 용어 | (사용자 노출 불필요, 삭제 권고) |
| 전자납부 | 어색한 한자어 합성어 | 온라인 결제 |
| 고지 | 행정어 | 안내 / 통보 |
| 처분 결과 | 법률 용어 | 심사 결과 |
| 유효기간 | 행정어 | 사용 가능 기간 |
| 법정 처리기한 | 법률 용어 | 법으로 정한 처리 기한 |
| 주민등록등본 | 공식 서류명(유지) → 용도 설명 누락 | 주민등록등본 (가족 구성원·주소 확인 서류) |
| 주민등록초본 | 공식 서류명(유지) → 용도 설명 누락 | 주민등록초본 (주소 이력 확인 서류) |

**허용 예외 원칙**: 공식 서류명(주민등록등본, 가족관계증명서 등)은 원어 유지 필수.  
단, 선택 화면에 괄호 병기로 용도를 설명해야 합니다.

---

### 2.2 정보핵심화의 원칙 위반 사례

**발견 위치**: 민원 처리 결과 안내, 접수 완료 화면, 발급 완료 화면

| 현재 표현 | 문제 | 권고 방향 |
|---------|------|---------|
| "민원 신청이 정상적으로 접수되었습니다. 처리 과정에서 담당 기관으로부터 추가 서류를 요청받을 수 있습니다. 처리 기간은 기관별로 상이할 수 있습니다." | 핵심(접수번호·처리기한)이 뒤에 있음 | 결론+접수번호+처리기한을 첫 줄에 |
| 발급 완료 화면에서 "파일 형식을 선택하시고 다운로드 버튼을 클릭하신 후…" | 단계 설명이 행동보다 앞에 있음 | "[PDF 내려받기] [화면 출력]" 버튼 먼저, 설명은 아래 |
| 인증 선택 화면: 3~4개 인증 방법 나열 후 설명 | 추천 방법이 없어 선택 부담 가중 | "간편인증 (카카오·네이버)으로 빠르게 확인하거나 다른 방법을 선택하세요." |

---

### 2.3 심리적안전망의 원칙 위반 사례

**발견 위치**: 에러 화면, 인증 실패, 서비스 점검 안내

| 현재 표현 | 문제 | 권고 방향 |
|---------|------|---------|
| "인증서 오류가 발생했습니다. 다시 시도해 주세요." | 원인 없음, 해결책 없음 | 원인(인증서 만료/비밀번호 불일치) + 대안(간편인증) 제시 |
| "처리가 완료되지 않았습니다." | 무엇이 왜 완료되지 않았는지 불명확 | 구체적 실패 원인(수수료 미결제) + 해결 행동 안내 |
| "서비스를 일시적으로 이용할 수 없습니다." | 언제 재개되는지 불명확 | 점검 시간 + 긴급 대안(주민센터 방문) 포함 |

## 3. 주요 문제 유형 요약

| 원칙 | 빈도 | 대표 위치 |
|------|------|---------|
| 무번역 위반 | ★★★ 높음 | 서비스 분류명, 법정 서식 레이블, 결과 화면 |
| 정보핵심화 위반 | ★★ 중간 | 완료 화면, 처리 현황, 도움말 |
| 심리적안전망 부재 | ★★ 중간 | 인증 실패, 서비스 오류, 처리 지연 |

## 4. Before / After 개선 예시

### E1 — 공동인증서 오류

```
🚫 현재:
"인증서 오류가 발생했습니다. 다시 시도해 주세요."

✅ 개선:
공동인증서 확인이 완료되지 않았습니다.
인증서 유효기간이 지났거나 비밀번호가 틀렸을 수 있습니다.
[간편인증으로 다시 시도하기]  [다른 인증 방법 보기]
```

> 적용 원칙: 심리적안전망 (원인 + 행동 제시), 무번역 (공동인증서 → 유지, 오류 이유는 일상어)

---

### E2 — 발급 미완료 (수수료 미결제)

```
🚫 현재:
"처리가 완료되지 않았습니다."

✅ 개선:
발급이 완료되지 않았습니다.
수수료가 결제되지 않아 서류가 발급되지 않았습니다.
[수수료 결제하기]
```

> 적용 원칙: 심리적안전망 (사용자 탓 없음, 구체적 원인 + 버튼 제시), 정보핵심화 (결론 첫 줄)

---

### E3 — 서비스 점검 안내

```
🚫 현재:
"서비스를 일시적으로 이용할 수 없습니다."

✅ 개선:
지금은 이 서비스를 이용할 수 없습니다.
서버 점검 중입니다. (점검 시간: 4월 19일 새벽 2시~6시)
점검이 끝난 뒤 다시 이용해 주세요.
급한 경우 가까운 주민센터를 방문해 주세요.
```

> 적용 원칙: 정보핵심화 (점검 시간 명시), 심리적안전망 (대체 경로 제공)

---

### E4 — 처리 기한 초과

```
🚫 현재:
"처리가 지연되고 있습니다."

✅ 개선:
처리 기한이 지났습니다.
담당 기관이 검토를 계속하고 있습니다.
접수번호({N})를 준비해 고객센터(1588-2188)나 담당 기관에 문의해 주세요.
```

> 적용 원칙: 정보핵심화 (접수번호 포함), 심리적안전망 (문의 경로 제공)

---

### E5 — 서류 첨부 오류

```
🚫 현재:
"첨부 파일 오류입니다."

✅ 개선:
파일을 첨부하지 못했습니다.
PDF, JPG, PNG 형식의 파일만 첨부할 수 있으며, 크기는 10MB 이하여야 합니다.
파일을 확인하고 다시 첨부해 주세요.
```

> 적용 원칙: 무번역 (형식·크기 조건 명확히), 심리적안전망 (사용자 탓 없음, 해결책 제시)

## 5. 기관 맞춤 톤·매너 권고

### 톤 기준

| 상황 | 권고 톤 | 샘플 |
|------|---------|------|
| 서비스 안내·설명 | T2 길잡이 | "다음 방법 중 하나로 본인을 확인해 주세요." |
| 발급·신청 완료 | T4 마침표 | "주민등록등본 발급이 완료됐습니다. 유효 기간: 3개월" |
| 오류·지연 | T1 해결사 | "처리 기한이 지났습니다. 고객센터(1588-2188)에 문의해 주세요." |
| 처리 중 | T3 중계자 | "{담당 기관}에서 서류를 검토하고 있습니다. 처리 기한: {날짜}" |

### 정부24 전용 규칙

1. **서비스명 포함 필수**: "발급이 완료됐습니다" (✗) → "주민등록등본 발급이 완료됐습니다" (✓)
2. **고령층 배려**: 핵심 정보(버튼·완료 사실·접수번호)는 반드시 첫 화면에 노출
3. **공식 서류명 + 용도 설명 병기**: "주민등록등본 (가족 구성원·주소 확인 서류)"
4. **막연한 약속 금지**: "빠른 시일 내 처리됩니다" (✗) → 정확한 날짜/기간 제시

## 6. 자가진단 체크리스트 — 정부24 맞춤

**공통 항목 (Layer 2 상속)**
```
□ 법령 용어·서식명 첫 등장 시 괄호 안에 일상어 설명이 있는가?
□ 에러 메시지에 사용자를 탓하는 표현이 없는가?
□ 민원 완료 화면에 접수번호와 처리 기한이 명시됐는가?
□ 버튼 레이블이 동사형 또는 관용적 허용어인가?
□ 결론이 첫 문장에 있는가?
□ "빠르게", "곧" 같은 막연한 약속이 없는가?
```

**정부24 전용 항목**
```
□ 발급 서류의 유효기간이 완료 화면에 안내됐는가?
□ 인증 실패 시 대체 인증 방법이 안내됐는가?
□ 처리 진행 중 화면에 담당 기관명과 처리 기한이 명시됐는가?
□ 서비스 점검 안내에 점검 시간과 대체 경로(주민센터 방문 등)가 포함됐는가?
□ 고령층을 위해 핵심 정보(버튼, 완료 사실, 접수번호)가 첫 화면에 노출됐는가?
□ 안내 문구에 서비스명이 명시됐는가? (예: "발급" → "주민등록등본 발급")
```
