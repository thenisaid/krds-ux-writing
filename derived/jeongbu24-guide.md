# 정부24 — UX Writing 심층 분석 및 파생 가이드 (전체 하위 페이지)

> **기관**: 정부24 (행정안전부, https://plus.gov.kr/)
> **분석 기준**: KRDS UX Writing 3대 원칙 (무번역·정보핵심화·심리적안전망)
> **Layer 2 트랙**: 표준형 (중앙행정기관 직영, 정부 상징 로고 사용)
> **주요 사용자**: 전 국민 — 영유아 보호자부터 고령층까지 가장 넓은 스펙트럼
> **탐색 대상**: 메인·민원서비스·혜택알리미·생활·정책정보·고객센터·원스톱·로그인 흐름
> **작성 기준**: IND-145 | 2026-04-24 (전면 재작성)

---

## 1. 기관 개요 및 서비스 맥락

정부24는 3,000여 개 민원·행정 서비스의 단일 진입점입니다.
전 국민이 사용자이므로 UX Writing은 최소 14세 이상이 직관적으로 이해할 수 있어야 합니다.

### 1.1 주요 서비스 구조

| 섹션 | URL 패턴 | 핵심 과업 |
|------|---------|---------|
| 민원서비스 | `/search/applyMw` | 민원 신청·조회·발급 |
| 혜택알리미 | `/portal/benefitV2` | 맞춤 혜택 조회·알림 설정 |
| 생활가이드 | `/portal/lgd` | 생애주기별 행정 가이드 |
| 정책정보 | `/portal/gvrnPolicy` | 정책뉴스·연구보고서·간행물·정책자료 |
| 고객센터 | `/portal/ntcmttr` | 공지사항·FAQ·이용안내 |
| 원스톱 서비스 | `/portal/onestopSvc/main` | 전입신고+·행복출산 등 복합 서비스 |
| 로그인 | `/member/login` | 공동인증서·간편인증·문자인증 |

### 1.2 사용자 스펙트럼

정부24는 단일 서비스 중 사용자 연령 범위가 가장 넓습니다.

| 사용자 그룹 | 비율 | UX Writing 함의 |
|-----------|------|---------------|
| 1020세대 | 약 15% | 간결함, 모바일 친화 |
| 3040세대 | 약 35% | 과업 효율, 정확한 상태 정보 |
| 5060세대 | 약 30% | 명확한 레이블, 큰 텍스트 지원 |
| 70세 이상 | 약 20% | 최소 전문어, 대체 경로 필수 |

> **레이어 원칙**: 고령층 기준으로 쓰면 모든 연령이 이해한다.

| 전입신고+ | 한번에 여러 감면 신청 | 한번에 여러 감면 신청 | ✓ (일치) |

#### 전입신고+ 설명문 문제

```
현행 (실측 텍스트):
거주지를 이동할 때는 주소지 변경・등록을 위해 전입한 날로부터 14일 이내에
새로운 거주지 관할 기관에 신고해야 합니다.
전입신고+는 관할 기관에 직접 방문하는 번거로움 없이 온라인으로 간편하게
신청할 수 있으며 신고자가 요금감면 대상자인 경우 신청서 작성 시
'사회적 배려대상자를 위한 요금감면 통합신청 동의'를 선택하면
전입신고와 함께 요금감면 신청까지 한 번에 일괄 신청되는 서비스입니다.
```

문제: 1문장이 80자 이상, 법률 용어('관할 기관', '사회적 배려대상자'), 서비스 이점이 뒤에 있음

#### 버튼 레이블 문제

| 현행 | 위치 | 문제 | 권고 |
|-----|------|-----|-----|
| `신청하기` | 전입신고+ 상세 | 주체·대상 없는 범용 레이블 | 전입신고 신청하기 |

1. **서비스명 포함 필수**: "발급이 완료됐습니다" (✗) → "주민등록등본 발급이 완료됐습니다" (✓)
2. **고령층 배려**: 핵심 정보(버튼·완료 사실·접수번호)는 반드시 첫 화면에 노출

4. **막연한 약속 금지**: "빠른 시일 내 처리됩니다" (✗) → 정확한 날짜/기간 제시
5. **접수번호·처리기한은 완료 화면 최상단**: 사용자가 스크롤 없이 볼 수 있어야 함
6. **단일 명칭 원칙**: 같은 서비스는 GNB·브레드크럼·카드·h2 모두 동일 명칭

```
3. "민원인이 제출하지 않아도 되는 서류(본인정보제공 요구)"
```

- 2번과 3번의 차이가 미묘하고 사용자가 구분하기 어려움
- 법령 근거(민원처리에 관한 법률 제10조, 제10조2)를 본문에 직접 인용 → 정보핵심화 위반

### E61: "내용보기" 6개 동일 링크 텍스트 ★★ [원칙 B — 정보핵심화]

**위치**: 혜택알리미 > "지금 많이 찾는 혜택" 캐러셀 6개 슬라이드 각각

**문제**: 6개 혜택 카드 모두 동일한 링크 텍스트 "내용보기" 사용. 스크린리더 사용자가 링크 목록을 탐색할 때 "내용보기" 6개가 연속으로 읽혀 어느 카드의 링크인지 구분 불가. WCAG 2.4.6 Headings and Labels 기준 위반.

> [창업 지원 혜택 보기]

또는 `aria-label`로 맥락 제공:

### E173 — 알림서비스 설정 "문자(SMS)", "카카오 알림톡", "이메일" 채널 목록 — 수신 시점·내용 안내 없음 ★★ [B]

### E175 — "연말정산 간소화 서비스 자료 조회" — '간소화'의 의미와 수동 추가 필요 항목 안내 없음 ★★ [A/B]

- **문제**: 두 인증 방식의 차이(간편인증=카카오·네이버·PASS 등, 공동인증서=기존 공인인증서)가 메뉴명에서 드러나지 않음. 어느 방식이 본인에게 맞는지 판단 기준이 없어 잘못된 선택 후 재시도 빈도 높음. "공동인증서"라는 새 명칭이 이전 "공인인증서"와 동일한지 처음 보는 사용자는 알기 어려움. "간편"이라는 단어만으로 어떤 앱·서비스로 로그인 가능한지 예측 불가
- **원칙**: B — 정보핵심화 (인증 방식 특성·사용 가능 앱 선제 표시)
- **권장 개선안**: "카카오·네이버·PASS 간편 로그인" / "공동인증서 (구 공인인증서)로 로그인" — 실제 사용 수단과 이전 명칭을 병기

### E182 — 생애주기 서비스 "취업/창업·학교·군입대·결혼·사망·출생" 나열 메뉴 — 상황 중심 진입 기준 부재 ★★ [B]
- **위치**: 메인 > 생애주기별 서비스
- **원문**: "취업/창업", "학교", "군입대", "결혼", "사망", "출생"
- **문제**: 생애 이벤트 중심 메뉴 구성은 좋지만 각 카테고리 진입 후 어떤 민원 서비스가 있는지 메뉴명에서 전혀 예측 불가. "결혼"을 클릭했을 때 혼인신고뿐 아니라 주소 이전·건강보험 변경·성명 변경 등이 포함되는지 알 수 없음. 중복 해당 상황(결혼 후 이사하면 "결혼"인지 "이사/전입"인지)에서 선택 어려움. 서비스 수를 병기하지 않아 카테고리 내 콘텐츠 밀도 파악 불가
- **원칙**: B — 정보핵심화 (카테고리 내 주요 서비스 2-3개 미리 표시)
- **권장 개선안**: 각 카테고리 아래 대표 서비스 예시 표시. 예: "결혼 — 혼인신고·이름 변경·건강보험 피부양자 등록 (7개 서비스)"

### E183 — 신청서 작성 화면 필수 항목 표시 "*" 단독 — 범례 안내 없음 ★★ [C]
- **위치**: 각종 민원 신청서 > 입력 양식 화면
- **원문**: 필드 레이블 옆 빨간 별표 "*"만 표시
- **문제**: 빨간 별표가 필수 항목을 의미함을 처음 보는 사용자는 알 수 없음. 화면 어느 위치에도 "* 필수 항목" 범례 표시 없음. 일부 필드는 조건부 필수(외국인등록번호는 외국인에게만 필수)인데 조건 설명 없음. 모바일 화면에서 빨간 별표가 잘 보이지 않아 접근성 문제도 있음
- **원칙**: C — 심리적 안전망 (필수 여부 기준 명시, 누락 방지)
- **권장 개선안**: 양식 상단에 "* 표시는 반드시 입력해야 하는 항목입니다" 안내 문구 추가. 조건부 필수 항목은 "(해당자만 필수)" 괄호 표기

### E184 — 신청 완료 화면 "민원이 접수되었습니다" 단독 메시지 — 다음 행동 안내 없음 ★★ [B/C]
- **위치**: 민원 신청 > 완료 화면
- **원문**: "민원이 접수되었습니다."
- **문제**: 접수 완료 후 처리 기간·결과 확인 방법·추가 제출 서류 여부 등 사용자가 즉시 알아야 할 정보가 없음. 접수번호는 표시되지만 이 번호로 무엇을 할 수 있는지 안내 없음. 처리 결과를 이메일·문자로 받으려면 어디서 설정하는지 링크도 없음. 다음 행동(며칠 후 결과 확인, 추가 서류 제출 방법)이 완전히 누락
- **원칙**: B — 정보핵심화 + C — 심리적 안전망 (완료 후 불안 해소, 다음 행동 안내)
- **권장 개선안**: "신청이 완료됐습니다. 처리까지 약 [N]일이 소요됩니다. [나의 민원 확인하기] [알림 설정하기]" — 예상 기간과 다음 행동 버튼 포함

### E185 — 앱 유도 배너 "정부24 앱으로 더 편리하게" — 앱 차별화 기능 미명시 ★ [B]
- **위치**: 모바일 웹 > 앱 다운로드 유도 배너
- **원문**: "정부24 앱으로 더 편리하게 이용하세요."
- **문제**: 앱의 어떤 기능이 모바일 웹보다 편리한지 구체 설명 없음. "편리하다"는 형용사만으로 다운로드 동기 부여 불충분. 앱에서만 가능한 기능(모바일 신분증, 알림, 간편인증 연동 등)을 강조하지 않음. 이미 앱을 설치한 사용자에게도 항상 노출되어 불필요한 마찰 유발
- **원칙**: B — 정보핵심화 (앱 차별화 기능 명시, 설치 여부 분기)
- **권장 개선안**: "앱에서는 모바일 신분증 발급·알림 수신·간편인증이 가능해요. [앱 열기] [앱 다운로드]" — 차별화 기능 명시, 설치 여부에 따른 버튼 분기

### E186 — 로그아웃 완료 화면 "로그아웃 되었습니다" 단독 안내 — 재진입 경로 없음 ★★ [C]

- **원칙**: C — 심리적 안전망 (완료 후 방향 제시)
- **권장 개선안**: "로그아웃 됐습니다. [메인으로 가기] [다시 로그인하기]" — 두 가지 다음 행동 버튼 제공

### E187 — 마이페이지 "알림 설정" — 어떤 알림을 받을 수 있는지 유형별 설명 없음 ★★ [B]
- **위치**: 마이페이지 > 알림 설정 화면
- **원문**: "알림 설정"
- **문제**: 알림 설정 메뉴에 진입해도 어떤 종류의 알림이 존재하는지 미리 표시되지 않음. 민원 처리 결과·혜택 안내·생애주기 이벤트 알림 등 유형이 다양하지만 토글 스위치만 나열되어 각 항목이 무엇을 의미하는지 한 줄 설명조차 없음. 알림을 전부 켰을 때 수신 빈도나 채널(이메일·문자·앱 푸시)이 어떻게 다른지도 안내 없음. 고령 사용자가 불필요한 알림을 받거나 중요 알림을 꺼버리는 설정 오류 발생
- **원칙**: B — 정보핵심화 (알림 유형·채널·수신 빈도 사전 안내)
- **권장 개선안**: 각 알림 항목 옆에 한 줄 설명 추가. 예: "민원 처리 결과 알림 — 신청한 민원이 처리되면 이메일과 문자로 알려드립니다". 상단에 "채널 선택" 섹션 분리(이메일 / 문자 / 앱 알림)

### E188 — "민원 처리 상태" 단계 표시 — 현 단계 의미·예상 남은 기간 안내 없음 ★★ [B/C]
- **위치**: 나의 민원 > 처리 현황 > 처리 단계 표시 바
- **원문**: "접수 완료 → 처리 중 → 완료" (단계 표시만)
- **문제**: 처리 단계 바는 있지만 "처리 중"이 구체적으로 어느 단계인지(담당자 배정·검토 중·결재 대기 등) 안내 없음. 예상 완료일이나 남은 처리 기간이 없어 사용자가 민원 처리가 지연되고 있는 건지 정상 범위인지 판단 불가. 처리 기간이 법정 기한(7일·14일 등)임을 안내하지 않아 기한 초과 민원을 직접 문의하지 못하는 경우 발생
- **원칙**: B — 정보핵심화 + C — 심리적 안전망 (현 단계 설명, 예상 기간 안내, 기한 초과 시 대응 방법)
- **권장 개선안**: 단계 표시 아래 한 줄 설명. "현재: 담당자 검토 중 (접수일로부터 3일 경과 / 처리 기한: 7일)". 기한 초과 시 "처리 기한이 지났습니다. 1544-9000으로 문의하거나 [민원 재촉구] 버튼을 이용하세요" 자동 안내

### E189 — 혜택알리미 "맞춤 혜택 추천" 필터 — 입력 정보와 추천 결과의 연결 관계 불명확 ★★ [B]
- **위치**: 혜택알리미 > 맞춤 혜택 추천 > 조건 입력 화면
- **원문**: "생애주기", "가구 유형", "관심 분야" 필터 드롭다운
- **문제**: 조건을 입력해 혜택을 추천받는 구조이지만 어떤 조건이 어떤 혜택에 영향을 주는지 전혀 안내되지 않음. 예를 들어 "취업 준비생"을 선택하면 어떤 종류의 혜택이 나오는지 예측 불가. 조건을 잘못 입력해 혜택이 적게 나온 경우 재설정 방법도 안내 없음. 추천 결과가 수백 건에 달해도 필터 기준이 없어 탐색 부담 과중
- **원칙**: B — 정보핵심화 (조건-혜택 연결 관계 가시화, 주요 혜택 예시 제공)
- **권장 개선안**: 조건 선택 시 "이 조건으로 찾을 수 있는 혜택 예시: 청년 월세 지원, 취업 성공 패키지, 국민취업지원제도 등 약 47개" 미리보기 표시. 추천 결과 상단에 "입력 조건 기준: 20대·취업준비생·수도권" 요약 표시

### E190 — 서비스 이용 약관 동의 화면 — 필수·선택 동의 항목 간 차이 설명 없음 ★★ [B/C]
- **위치**: 회원가입 > 서비스 이용 약관 동의 화면
- **원문**: 약관 목록 + "전체 동의" 체크박스 + 필수/선택 표시
- **문제**: 필수 동의 항목과 선택 동의 항목이 나열되지만 선택 동의를 거부하면 어떤 서비스가 제한되는지 안내 없음. "개인정보 처리방침", "위치기반서비스 이용약관" 등 제목만 있고 실제로 어떤 정보가 어떻게 수집·활용되는지 한 줄 요약이 없음. 약관 전문은 수십 페이지로 실질적으로 읽기 어려운 구조. 만 14세 미만 아동의 법정대리인 동의 절차도 상황 발생 시에만 표시되어 해당 사용자 혼란 발생
- **원칙**: B — 정보핵심화 + C — 심리적 안전망 (필수·선택 동의 결과 사전 안내)
- **권장 개선안**: 각 동의 항목에 한 줄 설명 추가. "선택 동의 항목 거부 시: 맞춤 혜택 추천 서비스 이용 불가". 약관 제목 옆 "요약 보기" 버튼으로 핵심 내용 3줄 요약 제공

### E191 — "공공데이터 활용 신청" — 신청 자격·승인 기간·API 활용 방법 사전 안내 없음 ★★ [A/B]
- **위치**: 정책정보 > 공공데이터 > 공공데이터 활용 신청
- **원문**: "공공데이터 활용 신청"
- **문제**: "공공데이터 활용 신청"은 개발자·기업·연구자 등 특정 사용자 대상 서비스이지만 메뉴명만으로는 일반 시민 서비스인지 전문 사용자 대상인지 구분 불가. 신청 자격(개인·법인·연구기관 별 조건), 데이터 형식(JSON·CSV·XML), 승인 기간(3~7 영업일), API 키 발급 절차가 신청 전 화면에 전혀 안내되지 않음. 일반 시민이 잘못 진입 후 이탈하거나 개발 지식 없이 신청 후 활용 불가 상황 발생
- **원칙**: A — 무번역 (전문 용어 → 대상·용도 중심 설명) + B — 정보핵심화 (신청 자격·형식·기간 사전 제공)
- **권장 개선안**: "행정 데이터 개방 신청 (개발자·연구자용)". 진입 첫 화면: "이 서비스는 앱·웹 개발이나 연구 목적으로 정부 데이터를 활용하려는 분을 위한 것입니다. 신청 후 3~7 영업일 내 API 키를 발급해 드립니다." 대상별 탭 분리(개인·법인·공공기관)

### E192 — "비회원으로 서비스 이용" — 이용 가능한 서비스 범위 안내 없음 ★★ [B]
- **위치**: 로그인 화면 > "비회원 이용" 링크
- **원문**: "비회원으로 이용하기"
- **문제**: 비회원으로 이용할 수 있는 서비스 범위가 명시되지 않음. 회원 로그인 없이 민원 신청·발급·조회 중 어느 범위까지 가능한지 알 수 없어 비회원 선택 후 중간 단계에서 로그인 요구로 이탈하는 경우 빈번. 비회원 이용 시 저장·기록이 되지 않는다는 경고도 없어 작성 내용을 잃는 사례 발생. "비회원 이용" 선택 시 어떤 인증 수단으로 본인 확인을 하는지도 안내 없음
- **원칙**: B — 정보핵심화 (비회원 이용 가능 범위 선제 안내)
- **권장 개선안**: "비회원으로 이용하기 (일부 서비스만 가능)". 클릭 시 팝업: "비회원으로 이용 가능한 서비스: 민원 조회·증명서 출력(본인 확인 필요). 이용 불가: 신청 이력 저장·맞춤 혜택 추천·알림 서비스"

### E193 — 발급된 증명서 유효기간 안내 없음 ★★★ [B/C]
- **위치**: 민원서비스 > 증명서 발급 > 발급 완료 화면
- **원문**: "발급이 완료되었습니다."
- **문제**: 증명서가 발급된 후 해당 문서의 유효기간(주민등록등본 3개월, 가족관계증명서 3개월, 납세증명서 1~3개월)이 발급 완료 화면에 표시되지 않음. 사용자는 오래된 발급본을 제출해 서류 반려 후 재발급 비용 및 시간을 낭비함. 특히 반복 제출이 필요한 금융·입학·취업 서류에서 혼란 빈번. 발급한 날짜만 표시되고 "몇 개월 이내 발급본"이라는 유효 기준이 없어 제출 기관 요건 충족 여부 스스로 판단 불가
- **원칙**: B — 정보핵심화 + C — 심리적 안전망 (유효기간 선제 안내, 재발급 오류 방지)
- **권장 개선안**: 발급 완료 화면에 "이 증명서는 발급일로부터 [N]개월 이내에 제출해야 합니다" 문구 추가. 제출처별 유효기간이 다른 경우 "제출 기관에 확인 필요" 안내. 발급 목록 화면에서 각 문서 옆에 유효기간 배지 표시

### E194 — 시스템 정기 점검 시간 사전 안내 없음 ★★ [C]
- **위치**: 정부24 메인 · 서비스 이용 불가 화면
- **원문**: "서비스 이용이 일시적으로 중단되었습니다."
- **문제**: 정부24는 매주 화요일 새벽 등 정기 점검 시간이 있으나 점검 시작 전 로그인 화면이나 메인 배너에 사전 안내가 없음. 점검 중 접속 시 단순 오류 메시지만 표시되고 점검 종료 시간이 없어 "언제 다시 이용 가능한가"를 알 수 없음. 긴급 서류 제출 기한이 있는 사용자(입찰 참가, 학교 등록)가 점검 시간과 겹쳐 심각한 피해를 입는 사례 발생. 점검 종료 후 알림 서비스도 없음
- **원칙**: C — 심리적 안전망 (점검 일정 사전 공지, 복구 시간 명시, 불안 해소)
- **권장 개선안**: 점검 24시간 전부터 메인 배너에 "○월 ○일 ○시~○시 시스템 점검 예정" 공지. 점검 화면에 종료 예정 시각과 긴급 대안(전화 민원, 방문 접수) 안내. 점검 전날 마이페이지 즐겨찾기 사용자에게 SMS/앱 푸시 옵션 제공

### E195 — 민원 첨부파일 형식·용량 제한 안내 없는 업로드 화면 ★★ [B]
- **위치**: 민원 신청 > 첨부파일 업로드 화면
- **원문**: "파일을 첨부하세요"
- **문제**: 첨부파일 업로드 버튼만 있고 허용 파일 형식(PDF, JPG, HWP 등), 최대 용량(파일당 5MB, 전체 20MB 등), 허용 파일 수 제한이 업로드 시도 전 표시되지 않음. 허용되지 않는 형식(HEIC, WEBP 등)이나 초과 용량으로 업로드 시 에러 메시지가 뜨지만 그때서야 제한 조건을 알게 됨. 모바일에서 카메라로 촬영한 사진이 HEIC 형식이라 업로드 실패하는 사례 빈번. 변환 방법 안내도 없음
- **원칙**: B — 정보핵심화 (업로드 조건 사전 표시, 실패 전 차단)
- **권장 개선안**: 업로드 버튼 옆에 "PDF·JPG·HWP 가능 · 파일당 최대 5MB · 최대 3개" 한 줄 안내. HEIC 파일 선택 시 "iPhone 사진 파일은 JPG로 변환 후 업로드해 주세요 [변환 방법 보기]" 즉시 안내

### E196 — 전입신고 온라인 신청 가능 조건 안내 없음 ★★★ [B/C]
- **위치**: 서비스 > 주민 > 전입신고
- **원문**: "전입신고"
- **문제**: 전입신고는 온라인 신청이 가능한 경우(단독 전입, 본인만 이사)와 오프라인 방문이 필수인 경우(세대원 포함 전입, 임차인 전입)가 나뉘나 메뉴 진입 전 조건 안내 없음. 방문이 필요한 경우 온라인 신청까지 완료한 뒤 처리 불가 통보를 받고 주민센터를 재방문하는 사례 발생. 임차 계약서·집주인 동의서 등 추가 서류가 필요한 경우도 서류 목록 사전 안내 없음. 특히 이사일 당일 신고 필요성을 모르는 경우도 많음
- **원칙**: B — 정보핵심화 + C — 심리적 안전망 (온라인 가능 조건 선제 안내, 재방문 오류 방지)
- **권장 개선안**: "전입신고 — 온라인으로 가능한가요?" 사전 질문형 분기. "혼자 이사 왔어요 → 온라인 신청 가능" / "가족과 함께 이사 왔어요 → 주민센터 방문 필요". 필요 서류 목록(임대차계약서, 집주인 확인서 등)을 진입 전 체크리스트로 제공

### E197 — 국가보훈 대상자 연계 서비스 행정어 + 혜택 범위 안내 없음 ★★ [A/B]
- **위치**: 서비스 > 보훈·복지 > 국가보훈 서비스
- **원문**: "국가유공자 확인원", "보훈급여금 신청", "국가보훈대상자 서비스"
- **문제**: '국가유공자', '보훈대상자', '보훈급여금', '참전유공자', '고엽제후유(의)증' 등 법령 행정 용어가 메뉴명에 그대로 노출되어 본인이나 가족이 해당되는지 판단 불가. 각 메뉴에서 제공하는 혜택(의료비 지원, 교육지원, 취업지원 등)이 서비스 진입 전 요약 없이 클릭해야만 확인 가능. 해당 여부 확인을 위한 자가 판단 기준(공상 군경, 순직 유족 등 구분)도 없음
- **원칙**: A — 무번역 (법령 행정 용어 → 대상·혜택 중심 표현) + B — 정보핵심화
- **권장 개선안**: "국가를 위해 헌신한 분과 그 가족을 위한 혜택" 상위 카테고리. "내가 해당되는지 확인하기 → 3가지 질문으로 확인" 자가 진단 플로우. 각 서비스 옆에 "의료비 지원 · 교육비 지원 · 취업 우선 채용" 등 혜택 1줄 요약 추가

### E198 — 여권 재발급 신청 처리 기간·수령 방법 선택 화면 안내 없음 ★★ [B/C]
- **위치**: 서비스 > 외교·여행 > 여권 재발급 신청
- **원문**: "여권 재발급 신청"
- **문제**: 여권 재발급 신청 화면에서 일반 발급(3~4주)과 긴급 발급(3~5일, 추가 수수료 발생)의 차이 및 선택 기준 안내 없음. 수령 방법(방문 수령, 등기 우편)에 따른 소요 일수 차이도 표시되지 않음. 출국일이 임박한 사용자가 일반 발급 신청 후 출국일 전 수령 불가 사실을 뒤늦게 발견하는 사례 발생. 재발급 신청 필요 서류(기존 여권 반납 여부 등)도 사전 안내 없음
- **원칙**: B — 정보핵심화 + C — 심리적 안전망 (처리 기간 선제 안내, 출국일 기준 가이드)
- **권장 개선안**: 신청 진입 시 "출국 예정일이 있으신가요?" 분기 질문. "3주 이내 출국 → 긴급 발급 신청 필요 (추가 수수료 ○만 원)" / "3주 이후 출국 → 일반 발급 가능". 수령 방법별 예상 소요일 달력 시각화 제공
