# 홈택스 — UX Writing 분석 및 파생 가이드

> **기관**: 국세청 홈택스 (https://hometax.go.kr/)
> **분석 기준**: KRDS UX Writing 3대 원칙 (무번역·정보핵심화·심리적안전망)
> **Layer 2 트랙**: 표준형 (중앙행정기관 직영)
> **주요 사용자**: 세금 납부·신고 의무가 있는 전 국민, 사업자, 세무 대리인
> **작성 기준**: IND-136 | 2026-04-23

---

## 1. 기관 개요 및 서비스 맥락

홈택스는 국세청이 운영하는 세금 신고·납부·증명 발급 통합 시스템입니다.  
과업 복잡도가 높고 실수 시 가산세 등 실질적 불이익이 발생하는 고위험 서비스입니다.

| 서비스 유형 | 대표 과업 |
|------------|---------|
| 세금 신고 | 종합소득세, 부가가치세, 원천세 |
| 세금 납부 | 전자납부, 분납, 세금계산서 조회 |
| 증명서 발급 | 납세증명서, 소득금액증명, 사업자등록증명 |
| 환급 | 환급 신청, 환급 계좌 등록 |
| 전자세금계산서 | 발급, 조회, 수정 |

## 2. UX Writing 진단 — 3대 원칙 기준

### 2.1 무번역의 원칙 위반 사례

홈택스는 세법 전문 용어가 UI 텍스트에 그대로 사용되는 빈도가 매우 높습니다.  
일반 납세자(세무 대리인 없는 개인 사업자·직장인)의 혼란이 가장 큰 영역입니다.

| 현재 표현 | 문제 | 권고 표현 |
|---------|------|---------|
| 결정세액 | 세법 전문 용어 | 최종 납부 세금 |
| 과세표준 | 세법 전문 용어 | 세금을 계산하는 기준 금액 |
| 세액공제 | 세법 용어 (공제 = 깎아주는 것) | 세금 감면 |
| 가산세 | 법령 용어 | 추가 세금 (늦게 내거나 신고하지 않을 때 붙는 금액) |
| 원천징수 | 세법 전문 용어 | 월급에서 미리 떼는 세금 |
| 환급 | 일상어 수준이나 문맥 설명 필요 | 환급 (낸 세금을 돌려받는 것) |
| 기납부세액 | 세법 용어 | 이미 낸 세금 |
| 소득공제 | 세법 용어 | 소득에서 빼주는 금액 |
| 전자납부번호 | 행정 내부 용어 | 납부 번호 |
| 사업장 | 사무적 표현 | 사업 주소 / 사무실 주소 |
| 신고기한 | 행정어 | 신고 기한 |
| 과세기간 | 세법 전문 용어 | 세금 계산 기간 (예: 1월~6월) |
| 무실적 신고 | 행정어 | 매출이 없어도 하는 신고 |

**홈택스 특이 사항**:  
세법 조문 인용이 불가피한 레이블(예: "과세표준")은 원어 유지 후 인접 도움말(ⓘ)에 일상어 설명을 추가하는 방식을 권고합니다.

---

### 2.2 정보핵심화의 원칙 위반 사례

**발견 위치**: 신고 완료 화면, 납부 안내, 환급 신청 결과

| 현재 표현 | 문제 | 권고 방향 |
|---------|------|---------|
| 신고서 제출 후 안내문: "귀하의 종합소득세 신고서가 접수되었습니다. 신고 기한 경과 후 세무서에서 내용을 검토하여 결정세액이 확정되는 경우…" | 핵심(접수번호·납부 기한)이 긴 설명 뒤에 묻힘 | 접수번호 + 납부 기한 첫 줄 |
| 환급 안내 화면: "귀하의 세액이 기납부세액을 초과하지 않는 경우 환급이 발생할 수 있으며…" | 이중부정+조건문 중첩 | "낸 세금이 더 많으면 차액을 돌려드립니다." |
| 납부 방법 4개 나열 (인터넷뱅킹·가상계좌·카드·무통장) 후 각 설명 | 추천 없이 나열만 | "간편: 계좌이체로 바로 납부하기 / 다른 방법 보기" |

---

### 2.3 심리적안전망의 원칙 위반 사례

**발견 위치**: 신고 오류, 마감일 임박 알림, 가산세 경고

| 현재 표현 | 문제 | 권고 방향 |
|---------|------|---------|
| "신고기한이 지났습니다. 가산세가 부과됩니다." | 경고만 있고 해결 방법 없음 | 가산세 발생 사실 + 지금 신고하는 방법 안내 |
| "오류: 데이터를 불러올 수 없습니다." | 원인 없음, 사용자 행동 없음 | 원인(서버 연결 일시 중단) + 새로고침/고객센터 안내 |
| "신고서 작성 중 오류가 발생했습니다. 처음부터 다시 입력해 주세요." | 작성 내용 소실 + 사용자 탓 | 임시 저장 여부 안내 후 복구 경로 제시 |
| 마감 D-1 알림: "종합소득세 신고기한이 내일까지입니다." | 긴박감만 조성, 행동 링크 없음 | "내일이 마감입니다. 지금 바로 신고하기 → [신고서 작성하기]" |

## 3. 주요 문제 유형 요약

| 원칙 | 빈도 | 대표 위치 |
|------|------|---------|
| 무번역 위반 | ★★★ 매우 높음 | 세금 항목명, 신고서 레이블, 결과 화면 |
| 정보핵심화 위반 | ★★ 중간 | 신고 완료 화면, 납부 안내, 환급 안내 |
| 심리적안전망 부재 | ★★ 중간 | 오류 화면, 마감 경고, 가산세 안내 |

## 4. Before / After 개선 예시

### E1 — 종합소득세 신고 완료 화면

```
🚫 현재:
"종합소득세 신고서가 접수되었습니다.
결정세액은 세무서 검토 후 확정됩니다."

✅ 개선:
종합소득세 신고가 완료됐습니다.
접수번호: {N} | 납부 기한: 5월 31일

납부할 세금: 320,000원
[지금 납부하기]  [나중에 납부하기]

※ 납부 기한을 넘기면 추가 세금(가산세)이 붙습니다.
```

> 적용 원칙: 정보핵심화 (접수번호·납부기한 첫 줄), 무번역 (결정세액 → 납부할 세금, 가산세 설명 추가)

---

### E2 — 신고기한 경과 (가산세 발생)

```
🚫 현재:
"신고기한이 지났습니다. 가산세가 부과됩니다."

✅ 개선:
신고 기한이 지났습니다.
지금 신고하면 추가 세금(가산세)이 붙습니다.
빠를수록 가산세가 줄어듭니다.
[지금 신고하기]  [가산세 계산해보기]

문의: 국세상담센터 126
```

> 적용 원칙: 심리적안전망 (행동 유도, 문의 경로), 무번역 (가산세 일상어 설명), 정보핵심화 (행동 링크 포함)

---

### E3 — 환급 결과 안내

```
🚫 현재:
"귀하의 세액이 기납부세액을 초과하지 않는 경우 환급이 발생할 수 있으며,
환급 계좌로 지급됩니다."

✅ 개선:
이미 낸 세금이 더 많습니다. 차액을 돌려드립니다.
환급액: 48,000원
입금 예정일: 6월 30일 | 계좌: ***-***-1234

계좌 정보가 달라졌다면 [환급 계좌 수정하기].
```

> 적용 원칙: 무번역 (기납부세액 → 이미 낸 세금, 이중부정 제거), 정보핵심화 (핵심 수치 첫 줄)

---

### E4 — 서류 첨부 오류 (세금계산서 업로드)

```
🚫 현재:
"파일 처리 중 오류가 발생했습니다."

✅ 개선:
파일을 올리지 못했습니다.
XML 또는 ZIP 형식의 파일만 올릴 수 있으며, 크기는 20MB 이하여야 합니다.
파일을 확인하고 다시 올려 주세요.
도움이 필요하면 국세상담센터(126)로 문의해 주세요.
```

> 적용 원칙: 심리적안전망 (원인 + 해결책 + 연락처), 정보핵심화 (결론 첫 줄)

---

### E5 — 신고서 입력 중 세션 만료

```
🚫 현재:
"세션이 만료되었습니다. 처음부터 다시 시작하세요."

✅ 개선:
자리를 비운 사이 로그인이 끊겼습니다.
보안을 위해 30분 이상 사용하지 않으면 자동으로 로그아웃됩니다.
입력 중이던 내용은 임시 저장됐습니다. 다시 로그인하면 이어서 작성할 수 있습니다.
[다시 로그인하기]
```

> 적용 원칙: 심리적안전망 (임시 저장 안내로 불안 해소, 사용자 탓 없음)

## 5. 기관 맞춤 톤·매너 권고

### 톤 기준

| 상황 | 권고 톤 | 샘플 |
|------|---------|------|
| 세금 항목 안내 | T2 길잡이 + 중립 | "납부할 세금: 320,000원. 5월 31일까지 납부해 주세요." |
| 신고 완료 | T4 마침표 (절제) | "종합소득세 신고가 완료됐습니다. 접수번호: {N}" |
| 오류·기한 초과 | T1 해결사 | "신고 기한이 지났습니다. 지금 신고하면 가산세가 줄어듭니다." |
| 처리 대기 | T3 중계자 | "담당자가 신고 내용을 확인 중입니다. (처리 기한: 6월 30일)" |

### 홈택스 전용 규칙

1. **세금 용어 일상어화 + 도움말 병기**: 결정세액(ⓘ 최종 납부 세금) 형식 권고
2. **금액은 항상 명시**: "세금이 부과됩니다" (✗) → "추가 세금 35,000원이 붙습니다" (✓)
3. **기한은 구체적 날짜로**: "기한 내에" (✗) → "5월 31일까지" (✓)
4. **가산세 안내는 공포 조성 금지**: 사실(가산세 발생) + 행동(지금 신고) 세트로 제공
5. **국세상담센터 126** 모든 에러 화면에 포함

## 6. 자가진단 체크리스트 — 홈택스 맞춤

**공통 항목 (Layer 2 상속)**
```
□ 세법 전문 용어 첫 등장 시 괄호 또는 ⓘ 도움말로 일상어 설명이 있는가?
□ 에러 메시지에 사용자를 탓하는 표현이 없는가?
□ 에러 메시지에 해결 방법 또는 문의 연락처가 포함됐는가?
□ 결론이 첫 문장에 있는가?
□ 이중 부정이 없는가?
□ 막연한 약속("곧", "빠르게")이 없는가?
```

**홈택스 전용 항목**
```
□ 납부·환급 화면에 금액과 날짜(기한·입금일)가 명시됐는가?
□ 신고 완료 화면에 접수번호가 포함됐는가?
□ 가산세 경고 화면에 "지금 신고하기" 행동 링크가 있는가?
□ 세션 만료 시 임시 저장 여부를 안내했는가?
□ 모든 에러 화면에 국세상담센터(126) 연락처가 있는가?
□ 세금 항목명 옆에 금액이 병기됐는가? (텍스트만으로 이해 가능한가?)
```
