## 1. 서울시립미술관 (SeMA) — Layer 3 파생 가이드

> **기관 특성**: 서울시 산하 공립 미술관. 전시 관람·예약·교육 프로그램·소장품 열람이 핵심 서비스.
> **Layer 2 트랙**: 확장형 (독자 브랜드 아이덴티티 보유, 톤 재정의 허용)
> **UX Writing 진단 출처**: `sema_analysis_supplement.md`, IND-60 검토 결과

---

### Step 1. 기관 핵심 서비스 파악

#### 주요 사용자 흐름 (3가지)

**① 전시 탐색 → 관람 예약 → 현장 방문**

| 단계 | 핵심 화면 | UX Writing 주의 사항 |
|------|---------|-------------------|
| 탐색 | 전시 목록, 필터 | 전시 제목 / 작가명 / 기간 표기 일관성 |
| 상세 | 전시 소개 페이지 | 도슨트·해설사 예약 안내 문구 |
| 예약 | 관람 날짜·시간 선택 | 매진 / 대기 / 취소 에러 메시지 |
| 완료 | 예약 확인 이메일 | 접수번호·입장 방법 안내 |

**② 교육 프로그램 신청 → 결제 → 교육 참여**

| 단계 | 핵심 화면 | UX Writing 주의 사항 |
|------|---------|-------------------|
| 탐색 | 프로그램 목록 | 대상 연령/수준 명확화 |
| 신청 | 신청서 입력 폼 | 레이블·플레이스홀더 기준 준수 |
| 결제 | 결제 수단 선택 | 취소 환불 정책 안내 |
| 완료 | 신청 완료 화면 | 준비물·집합 장소 안내 포함 |

**③ 소장품 검색 → 상세 열람 → 자료 활용**

| 단계 | 핵심 화면 | UX Writing 주의 사항 |
|------|---------|-------------------|
| 검색 | 검색창 + 필터 | 검색어 없음 / 결과 없음 메시지 |
| 상세 | 작품 정보 페이지 | 저작권·출처 표기 방식 |
| 활용 | 자료 요청 / 다운로드 | 이용 조건 안내 |

### Step 3. 기관별 톤·매너 정의

**기관명**: 서울시립미술관 (SeMA)
**주력 톤**: ✅ T2 길잡이 (탐색·안내) + ✅ T4 마침표 (완료·성공)
→ T1 해결사(에러)와 T3 중계자(로딩)는 Layer 2 기준 그대로 적용

#### 기관 특화 톤 적용 기준

| 상황 | 적용 톤 | SeMA 적용 샘플 문장 | 금지 표현 |
|------|---------|-------------------|---------|
| 전시 탐색·설명 | T2 길잡이 + 문화친근 | "지금 서소문본관에서 열리고 있는 전시입니다." | "화려한 전시입니다!", "꼭 보세요!" |
| 예약·결제·취소 | T2 길잡이 | "관람 날짜와 인원을 선택해 주세요." | 감탄사, 이모지 |
| 관람 완료 확인 | T4 마침표 | "예약이 완료됐습니다. 당일 QR 코드로 입장해 주세요." | "즐거운 관람 되세요 😊", 칭찬·감탄 |
| 전시 안내 (도움말) | T2 길잡이 + 문화친근 | "전시 해설사(도슨트)가 매일 오후 2시에 안내합니다." | 과도한 경어("하여 주시기 바랍니다") |
| 긴급 공지 (임시 휴관) | T1 해결사 | "4월 19일(토) 서소문본관은 운영하지 않습니다. 다른 분관을 이용해 주세요." | 막연한 안심("곧 정상화됩니다") |
| 검색 없음 | T2 길잡이 | "'{검색어}'에 해당하는 작품을 찾지 못했습니다. 다른 검색어나 장르로 다시 찾아보세요." | "죄송합니다", 사용자 탓 표현 |
| 교육 신청 완료 | T4 마침표 | "교육 신청이 완료됐습니다. 수업 하루 전날 안내 문자를 드립니다." | "감사합니다!", 과도한 환영 |
| 프로그램 매진 | T2 길잡이 | "선택하신 날짜의 수업이 마감됐습니다. [대기 신청하기]" | "아쉽게도" (감정 표현) |

**SeMA 전용 톤 노트**:
- `~합니다` 표준형과 `~해요` 친근체 **혼용 금지** — SeMA는 `~합니다` 표준형 단일 사용
- 전시 제목·작가명은 원문 표기 유지 (한국어 번역 병기 권장)
- "관람객"은 SeMA 공식 대명사로 허용; "고객님" 금지

### Step 4. 주요 에러 시나리오 5개

> **3단 구조**: 상황(What) → 원인(Why) → 행동(How)
> **준수 원칙**: 사용자 탓 금지, 이모지·느낌표 금지, 해결책 필수

**E1 — 관람 예약 마감**

```
🚫 현재: "예약이 마감되었습니다."

✅ 개선:
선택하신 시간대의 예약이 마감됐습니다.
다른 날짜나 시간을 선택하거나, [대기 신청하기]를 눌러 취소 자리를 받을 수 있습니다.
```

**E2 — 교육 신청 기간 종료**

```
🚫 현재: "신청 기간이 종료되었습니다."

✅ 개선:
이 프로그램의 신청 기간이 끝났습니다.
다음 기수 일정은 [교육 일정 확인하기]에서 확인할 수 있습니다.
```

**E3 — 날짜 입력 형식 오류 (플레이스홀더 "YYYYMMDD")**

```
🚫 현재: "날짜를 올바르게 입력해 주세요." (또는 안내 없음)

✅ 개선:
날짜 형식이 맞지 않습니다.
숫자 8자리로 입력해 주세요. 예: 20260418
```

**E4 — 소장품 자료 다운로드 권한 없음**

```
🚫 현재: "권한이 없습니다."

✅ 개선:
이 자료는 로그인 후 내려받을 수 있습니다.
학술·연구 목적의 고해상도 자료는 [자료 이용 신청하기]에서 별도 신청해 주세요.
```

**E5 — 결제 실패**

```
🚫 현재: "결제에 실패했습니다. 다시 시도해 주세요."

✅ 개선:
결제가 완료되지 않았습니다.
카드사 또는 결제 수단 문제일 수 있습니다.
다른 카드로 다시 시도하거나, 문제가 계속되면 고객센터(02-2124-8800)에 문의해 주세요.
```

### Step 5. 자가진단 체크리스트 — SeMA 맞춤

> 부록 B 23개 항목에서 SeMA 서비스에 해당하는 항목 선택 + SeMA 전용 항목 추가

**공통 항목 (부록 B 상속)**

```
□ 행정어·전문용어가 없거나 처음 등장 시 괄호 병기했는가?
□ 에러 메시지에 사용자를 탓하는 표현이 없는가?
□ 에러 메시지에 해결 방법이 포함됐는가?
□ 버튼 레이블이 동사형 또는 관용적 허용어인가?
□ 플레이스홀더가 레이블을 대체하지 않는가?
□ 완료 메시지에 이모지·느낌표·칭찬 표현이 없는가?
□ 이중 부정이 없는가?
□ 결론이 첫 문장에 있는가?
```

**SeMA 전용 항목 (추가)**

```
□ 전시 제목과 작가명의 표기가 일관되는가? (국·영문 병기 기준 준수)
□ 도슨트·레지던시 등 전문용어 첫 등장 시 괄호 병기가 있는가?
□ 예약·결제 화면에서 취소·환불 정책이 명확하게 안내되는가?
□ 관람 날짜·시간이 "YYYY년 M월 D일 (요일) 오전/오후 HH시" 형식인가?
□ 검색 없음 화면에 대안 탐색 경로(장르·기간 필터 등)가 안내되는가?
```

## 2. 복지로 — Layer 3 파생 가이드

> **기관 특성**: 보건복지부 운영 복지 서비스 통합 포털. 복지 급여 신청·자격 확인·서비스 추천이 핵심.
> **Layer 2 트랙**: 표준형 (정부 상징 로고 사용 기관에 준하는 엄격 적용)
> **대상 사용자**: 기초생활수급자, 장애인, 노인, 아동 가족, 취약계층 — 디지털 리터러시 편차 큼

---

### Step 1. 기관 핵심 서비스 파악

#### 주요 사용자 흐름 (3가지)

**① 복지 서비스 자격 확인 → 신청 → 결과 확인**

| 단계 | 핵심 화면 | UX Writing 주의 사항 |
|------|---------|-------------------|
| 탐색 | 복지 서비스 검색·추천 | "해당 없음"/"대상 아님" 문구의 심리적 충격 최소화 |
| 자격 확인 | 모의 계산 결과 | 결과 해석 언어 — 숫자·기준 명확화 |
| 신청 | 신청서 입력 폼 | 법령 용어 포함 레이블의 일상어 전환 |
| 진행 중 | 심사 현황 조회 | 대기 이유·예상 기간 투명 공개 |
| 완료 | 결과 통보 (승인/미승인) | 미승인 시 사용자 탓 없는 언어 |

**② 급여 수급 신청 (기초생활·아동수당 등)**

| 단계 | 핵심 화면 | UX Writing 주의 사항 |
|------|---------|-------------------|
| 안내 | 지원 대상 안내 | 자격 기준 일상어화 (소득인정액 → 인정 소득) |
| 서류 | 구비 서류 목록 | 서류명 일상어 + 발급 방법 링크 |
| 제출 | 서류 첨부·제출 | 파일 형식·용량 제한 안내 |
| 결과 | 승인/미승인 통보 | 미승인 이유 명확화 + 이의신청 경로 |

**③ 찾아오는 복지 — 생애주기별 서비스 추천**

| 단계 | 핵심 화면 | UX Writing 주의 사항 |
|------|---------|-------------------|

### Step 3. 기관별 톤·매너 정의

**기관명**: 복지로 (보건복지부)
**주력 톤**: ✅ T2 길잡이 (자격 확인·신청 안내) + ✅ T1 해결사 (에러·거부 결과)
→ 취약계층 특성상 심리적 부담 최소화가 최우선 원칙

#### 기관 특화 톤 적용 기준

| 상황 | 적용 톤 | 복지로 적용 샘플 문장 | 금지 표현 |
|------|---------|------------------|---------|
| 서비스 자격 안내 | T2 길잡이 | "이 지원금은 소득 기준의 50% 이하 가구가 신청할 수 있습니다." | "혜택을 누리세요", 촉구 표현 |
| 신청서 입력 도움말 | T2 길잡이 | "세대원 전체의 주민등록상 주소를 입력해 주세요. (예: 서울시 종로구)" | 법령 조문 그대로 복사 |
| 모의 계산 결과 — 가능 | T4 마침표 | "계산 결과, 이 지원금을 신청할 수 있습니다. 실제 지원 여부는 심사 후 결정됩니다." | "축하드립니다!", 과도한 긍정 |
| 모의 계산 결과 — 불가 | T1 해결사 | "현재 소득 기준을 초과해 이 지원금 신청이 어렵습니다. 비슷한 지원금을 찾아보세요. [다른 지원금 보기]" | "죄송합니다", 사용자 탓 |
| 심사 진행 중 | T3 중계자 | "담당자가 서류를 확인하고 있습니다. 처리까지 약 30일이 걸립니다. 결과는 문자로 알려드립니다." | "곧 처리됩니다", 막연한 안심 |
| 승인 완료 | T4 마침표 | "지원금 신청이 승인됐습니다. 지원금은 {날짜}에 등록하신 계좌로 입금됩니다." | "잘하셨습니다!", 이모지 |
| 미승인 통보 | T1 해결사 | "이번 심사에서 지원 대상에 해당하지 않았습니다. 심사 기준: [소득 초과/부양의무자 기준]. 결과에 동의하지 않으면 [이의 신청하기]." | "유감입니다", 사용자 탓 |
| 제출 기한 임박 알림 | T2 길잡이 | "추가 서류 제출 기한이 3일 남았습니다. (기한: 4월 30일)" | 긴박감 조성 표현, 느낌표 |

**복지로 전용 톤 노트**:
- 미승인·거부 결과 화면에서 절대 사용자 탓 표현 사용 금지
- "축하"·"기쁩니다" 등 과도한 긍정 표현은 결과를 왜곡할 수 있으므로 금지
- 법령 조문 인용 시 반드시 `괄호 병기(일상어)` + `다음 줄 요약`을 추가
- 취약계층 특성상 모든 에러·거부 화면에 전화 문의 경로를 포함할 것 (129)

### Step 4. 주요 에러 시나리오 5개

**E1 — 미승인 (소득 기준 초과)**

```
🚫 현재: "심사 결과, 지원 대상에 해당되지 않습니다."

✅ 개선:
이번 심사에서 지원 대상에 해당하지 않았습니다.
이유: 소득인정액(인정 소득)이 기준 중위소득 50%를 초과했습니다.
결과에 동의하지 않으면 통보 후 90일 안에 이의를 신청할 수 있습니다.
[이의 신청하기]  [다른 지원금 찾기]
```

**E2 — 서류 형식 오류 (파일 업로드)**

```
🚫 현재: "파일 업로드에 실패했습니다."

✅ 개선:
파일을 올리지 못했습니다.
PDF, JPG, PNG 형식의 파일만 올릴 수 있으며, 파일 크기는 5MB 이하여야 합니다.
파일을 확인하고 다시 올려 주세요.
```

**E3 — 중복 신청 방지**

```
🚫 현재: "이미 신청된 내역이 있습니다."

✅ 개선:
이미 같은 지원금을 신청하셨습니다.
신청 현황은 [나의 복지] > [신청 내역]에서 확인할 수 있습니다.
추가로 문의하실 내용이 있으면 복지로 고객센터(129)로 전화해 주세요.
```

**E4 — 본인 인증 실패**

```
🚫 현재: "본인인증에 실패했습니다. 다시 시도해 주세요."

✅ 개선:
본인인증이 완료되지 않았습니다.
휴대전화 정보(번호·통신사)가 주민등록 정보와 일치하지 않을 수 있습니다.
다른 인증 방법을 선택하거나, 고객센터(129)에 문의해 주세요.
[다른 방법으로 인증하기]
```

**E5 — 세션 만료 (신청서 작성 중)**

```
🚫 현재: "세션이 만료되었습니다."

✅ 개선:
자리를 비운 사이 로그인이 끊겼습니다.
보안을 위해 30분 이상 사용하지 않으면 자동으로 로그아웃됩니다.
입력 중이던 내용은 임시 저장됐습니다. 다시 로그인하면 이어서 작성할 수 있습니다.
[다시 로그인하기]
```

### Step 3. 기관별 톤·매너 정의

**기관명**: 정부24 (행정안전부)
**주력 톤**: ✅ T2 길잡이 (민원 안내·서류 선택) + ✅ T4 마침표 (발급·신청 완료)
→ T1 해결사(오류)와 T3 중계자(처리 대기)는 Layer 2 기준 그대로 적용

#### 기관 특화 톤 적용 기준

| 상황 | 적용 톤 | 정부24 적용 샘플 문장 | 금지 표현 |
|------|---------|------------------|---------|
| 서비스 검색 결과 | T2 길잡이 | "찾으시는 서비스가 없으면 고객센터(1588-2188)로 문의해 주세요." | "다양한 서비스를 한 곳에서!" |
| 서류 발급 안내 | T2 길잡이 | "주민등록등본은 가족 구성원과 주소 이력을 확인하는 서류입니다." | 법령 조문 그대로 복사 |

| 처리 진행 중 | T3 중계자 | "{담당 기관}에서 서류를 검토하고 있습니다. 처리 기한: {날짜}." | "곧 처리됩니다" |

| 인증 실패 | T1 해결사 | "본인 확인이 완료되지 않았습니다. 인증 방법을 다시 선택하거나, 고객센터(1588-2188)에 문의해 주세요." | 사용자 탓 표현, 이모지 |

**정부24 전용 톤 노트**:
- 서비스 종류가 매우 많으므로 안내 문구는 반드시 해당 서비스명을 포함할 것 ("발급이 완료됐습니다" → "주민등록등본 발급이 완료됐습니다")
- 고령층 사용자 비율이 높아 화면 폰트·대비와 함께 텍스트 길이 최소화 원칙 강화
- 법정 서식 필드의 레이블은 공식 서류명을 유지하되, 인접한 도움말 텍스트에서 일상어 설명 추가

### Step 4. 주요 에러 시나리오 5개

**E1 — 공동인증서 오류**

```
🚫 현재: "인증서 오류가 발생했습니다."

✅ 개선:
공동인증서 확인이 완료되지 않았습니다.
인증서 유효기간이 지났거나 비밀번호가 틀렸을 수 있습니다.
인증서를 다시 선택하거나 [간편인증으로 로그인하기]를 이용해 주세요.
```

**E2 — 발급 기록 없음 (미납 수수료)**

```
🚫 현재: "처리가 완료되지 않았습니다."

✅ 개선:
발급이 완료되지 않았습니다.
수수료가 결제되지 않아 서류가 발급되지 않았습니다.
[수수료 결제하기]를 눌러 결제를 완료해 주세요.
```

**E3 — 서비스 점검 중**

```
🚫 현재: "서비스를 일시적으로 이용할 수 없습니다."

✅ 개선:
지금은 이 서비스를 이용할 수 없습니다.
서버 점검 중입니다. (점검 시간: 4월 19일 새벽 2시~6시)
점검이 끝난 뒤 다시 이용해 주세요.
급한 경우 담당 기관(읍·면·동 주민센터)을 방문해 주세요.
```

**E4 — 처리 기한 초과 (담당 기관 지연)**

```
🚫 현재: "처리가 지연되고 있습니다."

✅ 개선:
처리 기한이 지났습니다.
담당 기관이 검토를 계속하고 있습니다.
접수번호({N})를 준비해 고객센터(1588-2188)나 담당 기관에 문의해 주세요.
```

**E5 — 서류 첨부 오류 (형식·용량)**

```
🚫 현재: "첨부 파일 오류입니다."

✅ 개선:
파일을 첨부하지 못했습니다.
PDF, JPG, PNG 형식의 파일만 첨부할 수 있으며, 크기는 10MB 이하여야 합니다.
파일을 확인하고 다시 첨부해 주세요.
```
