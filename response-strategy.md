## KRDS와의 관계 — Depth Extension 프레임

KRDS 7개 원칙 중 5개가 선언 수준에 머문다. 원칙이 있지만 "어떻게"가 없다.

| 레이어 | 역할 | 주체 |
|--------|------|------|
| Layer 1 (선언) | 방향과 가치 선언 | KRDS 공식 |
| Layer 2 (실행) | 판별 기준 + 대체어 + 패턴 | **우리 가이드** |

**자기 모순 증거**: KRDS는 "동사형 레이블" 원칙을 선언하면서, 공식 사이트 내에 "더보기", "초기화" 같은 명사형 레이블을 그대로 사용한다. 우리 가이드는 이 간극을 실행 기준으로 채운다.

### KRDS 원칙 × 우리 원칙 매핑

| KRDS 공식 원칙 | 수준 | 우리 원칙의 실행 기여 |
|--------------|------|------------------|
| 원칙 2. 모든 사용자를 포용 | 선언 (물리적 접근성만) | **무번역** — 14세 기준 인지 접근성 수치화 |
| 원칙 4. 빠르고 간단 | 선언 (의사결정 최소화) | **정보핵심화** — 결론 우선, 단어 수 제한, 청킹 |
| 원칙 5. 쉽게 이해·사용 | 선언 (어떤 단어가 쉬운지 기준 없음) | **무번역** — 행정어 대체어 사전, 판별 기준 |
| 원칙 7. 신뢰 | 선언 (기관 신뢰만) | **심리적안전망** — 사용자 정서 신뢰, 에러 3단 구조 |
| 버튼 동사형 원칙 | 부분 실행 (버튼만, 자기 모순) | **심리적안전망** — 전체 UI 텍스트로 확장 |

## 대상별 커뮤니케이션 가이드

### UX 라이터

> "KRDS가 '쉽게 써라'고 선언한 자리에, 우리는 '이 단어 대신 이 단어를 쓰라'는 판별 기준을 놓습니다."

- 관심사: 실제 업무에서 바로 쓸 수 있는 기준
- 핵심 메시지: 원칙이 아닌 패턴. 사전, 체크리스트, 예시

### 정부 관계자 / 발주처

> "KRDS를 이미 따르고 계신다면, 이 가이드는 텍스트 레벨에서 감사 대응 가능한 기준으로 완성합니다."

- 관심사: KRDS 준수 여부, 책임 소재 명확화
- 핵심 메시지: 충돌이 아닌 보완. KRDS 위에 쌓이는 실행층

### 디자이너

> "KRDS 컴포넌트에 어떤 글자를 넣을지, 스펙처럼 참조할 수 있는 텍스트 기준입니다."

- 관심사: 컴포넌트 단위 적용 가이드
- 핵심 메시지: 버튼 텍스트, 에러 문구, 플레이스홀더 — 상황별로 바로 참조

## 10개 공백 × 3대 원칙 매핑

KRDS가 다루지 않는 10개 영역과 우리 3대 원칙의 연결:

| # | 공백 영역 | 적용 원칙 |
|---|---------|---------|
| 1 | 에러·실패 언어 원칙 | 심리적안전망 |
| 2 | 폼 레이블·플레이스홀더 | 무번역, 정보핵심화 |
| 3 | 안내문·도움말 톤앤매너 | 무번역, 심리적안전망 |
| 4 | 단어 난이도 기준표 | 무번역 |
| 5 | 정보 구조 원칙 | 정보핵심화 |
| 6 | 성공·완료 메시지 패턴 | 심리적안전망 |
| 7 | 빈 상태·검색 없음 텍스트 | 심리적안전망 |
| 8 | 상황별 톤 가이드 | 3대 원칙 전체 포괄 |
| 9 | 알림·토스트 메시지 | 정보핵심화, 심리적안전망 |
| 10 | 접근성 텍스트 (alt·aria-label) | 무번역, 포용 |

## 콘텐츠 아키텍처 선택지 비교

| 구조 | 방식 | 장점 | 단점 |
|------|------|------|------|
| **A. 원칙 우선형** | 3대 원칙을 챕터로, 각 챕터에 상황별 가이드 | 원칙 이해 → 응용 가능. 재사용성 높음 | 특정 상황 찾기 위해 챕터 이동 필요 |
| B. 상황 우선형 | UI 상황별 구성, 각 상황에 원칙 적용 | 실무 검색 빠름 (에러 상황 → 바로 해당 섹션) | 원칙 맥락 희석, 중복 발생 |
| C. KRDS 매핑형 | KRDS 공식 구조 따라 "선언 → 실행" 구조 | KRDS 대응 문서임을 명확히 전달 | KRDS 변경 시 종속성 발생 |

**권장: 선택지 A (원칙 우선형)**

이유: 단기 납품용 레퍼런스 카드(부록 C)와 상황별 톤 가이드(5장)로 실무 접근성 보완 가능. 원칙 이해 없이 패턴만 복사하는 문제를 방지.

## 권장 목차 (선택지 A — 원칙 우선형)

```
서문
  - 이 가이드가 필요한 이유
  - KRDS와의 관계 (Layer 1 선언 / Layer 2 실행)
  - 사용 방법

제1장. 파운데이션 [완성 ✅]
  1.1 비전: 시민이 읽는 글, 국가가 쓰는 말
  1.2 주권자: 이 가이드를 쓰는 사람 / 읽는 사람
  1.3 3대 원칙 개요 (무번역 / 정보핵심화 / 심리적안전망)
  1.4 보이스·톤 기준

제2장. 무번역의 원칙 — 실행 기준
  2.1 행정어·전문용어 대체어 사전 [완성 ✅ — 150개 항목, 6개 사이트 실사]
  2.2 폼 레이블·플레이스홀더 작성 기준
  2.3 접근성 텍스트 (alt·aria-label) 작성 원칙

제3장. 정보핵심화의 원칙 — 실행 기준
  3.1 안내문·설명문 결론 우선 구조 (역피라미드)
  3.2 정보 구조 원칙 (단어 수 제한, 청킹)
  3.3 알림·토스트·모바일 메시지 기준

제4장. 심리적안전망의 원칙 — 실행 기준
  4.1 에러 메시지 3단 구조 (무슨 일 / 왜 / 어떻게) [완성 ✅]
  4.2 성공·완료 메시지 패턴
  4.3 빈 상태·검색 없음 텍스트

제5장. 상황별 톤 가이드 (3원칙 통합)
  5.1 탐색·안내 상태
  5.2 대기·로딩 상태
  5.3 오류·실패 상태
  5.4 완료·성공 상태

제6장. → KRDS 컴포넌트 × 텍스트 매핑 [별도 파일 링크]
  (krds-component-text.md — 본문에서 링크로 이동)

부록
  A. 금지 표현 목록
  B. 자가진단 체크리스트
  C. 레퍼런스 카드 (즉시 사용 가능, 인쇄 가능)
```

## P1/P2/P3 실행 로드맵

| 단계 | 범위 | 산출물 |
|------|------|--------|
| **P1 즉시** | 2.1 행정어 사전 ✅ + 4.1 에러 메시지 ✅ + 부록 B 체크리스트 | 현장 즉시 적용 가능한 핵심 3종 |
| **P2 단기** | 3.1~3.2 안내문·정보구조 + 4.2~4.3 + 5장 전체 | 전체 상황 커버리지 완성 |
| **P3 중기** | 6장 컴포넌트 매핑 + 기관 파생(필요 시) | 확장 적용 기반 구축 |

P1은 행정어 사전 + 에러 구조 + 체크리스트 3종으로 시작한다. 이 세 가지는 가장 즉각적인 업무 영향력을 가지며, KRDS 공백 중 현장에서 가장 자주 마주치는 영역이다.

## 결정 사항 (확정)

### DECISION A — 제6장 컴포넌트 매핑 ✅ 확정

**별도 페이지로 분리 + 본문에서 링크 연결**

- 제6장은 본문에 포함하지 않음
- 본문 해당 위치에 "→ KRDS 컴포넌트 × 텍스트 매핑 보기" 링크 배치
- 컴포넌트 매핑은 별도 파일(`krds-component-text.md`) 또는 별도 웹 페이지로 관리
- 이유: 46개 컴포넌트 전체를 다루려면 본문 분량을 초과하므로, 플로우 상 링크로 이동하는 구조가 적합

### DECISION B — 기관 파생 처리 ✅ 확정

**완전 별도 파일 분리 — 이 문서의 범위 밖**

- 기관 파생(SeMA, 행안부 등)은 이 가이드라인의 범위가 아님
- **현재 작업 범위**: KRDS UX Writing Guidelines — KRDS 기반 범용 공공 서비스 텍스트 원칙
- 기관별 파생은 이 가이드를 완성한 뒤 별도 프로젝트로 진행
- 본문에 기관 파생 관련 내용 일체 포함하지 않음
