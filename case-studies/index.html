4.1절 — 에러 메시지 3단 구조

실전 Before / After 사례

공공 서비스 오류 메시지를 3단 구조로 바꾼 실례 3건.
각 사례가 적용한 KRDS 원칙을 태그로 확인하세요.

무번역의 원칙 — 일상어 사용
정보핵심화 — 구체적 정보 우선
심리적안전망 — 사용자를 탓하지 않음
3단 구조 공식:  1단 상황(What) — 무슨 일이 있었는지  |  2단 원인(Why) — 왜 그런지  |  3단 행동(How) — 지금 무엇을 하면 되는지. 단, 1단(상황)은 항상 필수. 2·3단은 맥락에 따라 축약 가능.
1

인증 번호 불일치

무번역의 원칙 심리적안전망
BEFORE — 기존 메시지
ERROR 4023: 인증 실패 인증에 실패했습니다. 다시 시도해 주세요.
AFTER — 3단 구조 적용
1단 상황 인증 번호가 일치하지 않습니다.
2단 원인 문자로 받은 6자리 숫자와 다르게 입력됐습니다.
3단 행동 문자를 다시 확인하고 숫자를 입력해 주세요.
번호가 만료됐다면 [인증 번호 다시 받기]를 눌러 주세요.
왜 바꿔야 하나? "ERROR 4023"은 시민에게 의미 없는 기술 코드입니다. "인증 실패 + 다시 시도"만으로는 무엇이 틀렸는지, 어떻게 해야 하는지 알 수 없습니다. 3단 구조는 원인(6자리 불일치)과 대안(번호 재발송)을 동시에 제공해 사용자가 스스로 해결할 수 있게 합니다.
2

파일 업로드 실패

정보핵심화 심리적안전망
BEFORE — 기존 메시지
파일 업로드에 실패했습니다. 오류가 발생했습니다. 잠시 후 다시 시도해 주세요.
AFTER — 3단 구조 적용
1단 상황 파일을 올리지 못했습니다.
2단 원인 파일 크기가 10MB를 넘으면 올릴 수 없습니다.
3단 행동 파일 크기를 줄이거나, 여러 개로 나눠 올려 주세요.
지원 형식: PDF, JPG, PNG / 최대 10MB
왜 바꿔야 하나? "오류가 발생했습니다"는 빈 메시지입니다. 원인도 해결책도 없어 사용자가 할 수 있는 것이 없습니다. 개선된 메시지는 10MB 제한이라는 핵심 정보를 2단에 배치해 사용자가 즉시 행동을 결정할 수 있게 합니다. 지원 형식까지 병기해 동일 오류 재발을 방지합니다.
3

세션 만료

무번역의 원칙 정보핵심화 심리적안전망
BEFORE — 기존 메시지
세션이 만료되었습니다. 로그인 페이지로 이동합니다.
AFTER — 3단 구조 적용
1단 상황 자리를 비운 사이 로그인이 끊겼습니다.
2단 원인 보안을 위해 30분 이상 사용하지 않으면 자동으로 로그아웃됩니다.
3단 행동 작성 중이던 내용은 임시 저장됐습니다. 다시 로그인하면 이어서 작성하실 수 있습니다.
[다시 로그인하기]
왜 바꿔야 하나? "세션 만료"는 IT 용어입니다. 시민은 "세션"이 무엇인지 모릅니다. 개선된 메시지는 일상어("로그인이 끊겼습니다")로 상황을 설명하고, 보안 정책(30분 자동 로그아웃)을 원인으로 명시합니다. 핵심은 임시 저장 안내 — 사용자가 가장 두려워하는 데이터 손실을 3단에서 먼저 해소해 심리적 안전망을 형성합니다.

에러 메시지 작성 핵심 원칙

❌
금지: "잘못 입력하셨습니다" · "올바르지 않은 정보입니다" · "오류가 발생했습니다(단독)" · "잠시 후 다시 시도해 주세요(단독)" · "ERROR [코드]" · "죄송합니다(오류 맥락)"
✅
필수: 1단 상황은 항상 포함. 사용자 탓 언어 대신 "일치하지 않습니다 / 확인이 필요합니다"로 중립 표현. 오류 메시지에 이모지·느낌표 금지.
💡
팁: 원인 불명 시 억지 이유 만들지 않기 — "일시적인 문제가 생겼습니다. 문제가 계속되면 고객센터(☎ 1234-5678)에 문의해 주세요."로 솔직하게.
